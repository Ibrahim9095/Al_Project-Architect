# AI Project Architect

# Project Classification

## 1. Purpose

Project Classification is the engineering process of determining the complexity, scale, documentation requirements, repository structure, and implementation strategy of a software project before any technical design begins.

Not every project requires the same engineering process.

A personal portfolio website should not produce the same documentation as an Enterprise ERP platform.

AI Project Architect automatically analyzes the project and selects the appropriate engineering level.

This document defines the classification methodology used throughout the platform.

---

# 2. Objective

The objectives of Project Classification are:

- Measure overall project complexity.
- Select the correct engineering workflow.
- Determine documentation depth.
- Determine repository structure.
- Select engineering templates.
- Estimate implementation effort.
- Activate domain knowledge.
- Prepare AI Coding Agents.
- Generate an appropriate export package.

Project Classification becomes the foundation for every engineering decision that follows.

---

# 3. Engineering Philosophy

AI Project Architect follows one important principle:

> **Every project deserves the engineering process it actually needs—not more, and not less.**

Under-engineering creates technical debt.

Over-engineering wastes time and resources.

The AI should always recommend the smallest engineering process capable of delivering a reliable solution.

- Remain objective.
- Avoid assumptions.
- Justify every classification decision.
- Prefer evidence over estimation.

# 8. Engineering Classification Levels

The selected level determines:

- Repository Structure
- Documentation Depth
- AI Discovery Strategy
- Architecture Complexity

- Engineering Standards
- Export Package

The objective is to apply the appropriate amount of engineering for each project.

Each dimension contributes to the final classification.

The generated repository becomes the official engineering workspace for the project.

# 43. Classification Lock

A project classification should remain editable only during Discovery.

- Regenerating Documentation
- Updating Repository Structure
- Recalculating Engineering Complexity
- Revalidating Dependencies

This prevents inconsistencies across generated engineering artifacts.
